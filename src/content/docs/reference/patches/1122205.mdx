### Code & Tuning Changes
<Aside>Please see the [forum post](https://forums.ea.com/blog/the-sims-game-info-hub-en/summary-of-updates-to-sims%E2%80%99-eye-blinking/13174182) for information on the eyelid fixes. </Aside>

##### General
- Created a boolean tuning field 'allow_transfer_out_of_hh_inventory' that, when enabled, will allow an object in the HH inventory that cannot be directly placed in-world (eg. livestock animals) to be properly slotted to an in-world object. 
    - Enabled this boolean in the tuning for the loot action that slots livestock animals to their sheds.
- Modified the function that removes the animal shed object to return early if the shed is currently in the HH inventory (so the animal shed will not get erroneously unregistered from the animal service when moving HH).
- Stopped tracking objects in the lost and found service if their location is currently in the Household Inventory (because they cannot be lost if they are in the HH inv).
- Properly set the livestock animals' owner ID field in the python code after they are assigned to a shed (since livestock animals cannot be placed in-world they were bypassing the logic that normally sets that field, causing the Sim to not own them anymore after moving HH).
- Objects duplicated between world and inventory will be discarded from inventory on load
- When scheduling prom, we were testing for any early graduate Sims, but the Tests would always fail on older saves. Moved those tests to when prom is run.
- Updated score multiplier test to return a default value instead of throwing an error when running on simless events. (In relation to Prom Goals not being completed.)
- Re-exported premade household info with pets in game to fix pets missing in premade households.
- Removing Unflirty tested outcomes from Mixer_social_Hug_targeted_Friendly_MiddleScore.
- Fixing a test on landlordTenant_DoInspectionMaintenance (In relation to "Do Inspection and Maintenance" previously not being available on broken objects for landlord sims.).
- Renamed some relbits and supporting descriptions in Parenthood. (In relation to Child and Teen sims getting the Super Sibling relationship, even if not related. This was implemented as a feature and the relationship renamed, instead of "fixed".)
- Added state smallBusiness_OpenClose_close to object_RetailPedestal (the tuning file for the Plant Containment Cell). (In relation to a Script Call Fail error when the Plant Containment Cell is on the lot and the player tries to place other objects.)

<Card title="Safeguards Against Inappropriate Sibling Relationships">
We've added safeguards in the game to check against inappropriate sibling relationships. When the game loads, the game will check to see if a Sim has any spouse or fiance, and if they do it will also check to see if they share parents. If any shared parents are found, the game will remove them from the Sim, and then reapply the correct relationship and remake the FamilyTree. This fix may not address all instances in which incorrectly available romantic actions can occur, but rather this specific kind. As always, please report any new instances on the forums so we can get them properly logged and accounted for in upcoming fixes.

- After Load, the GenealogyTracker has a fix to remove shared parents from Sims that are married/engaged now.
- When all Sims are fully loaded, the Genealogy Tracker checks if they have any spouses or fiancees, and if they share any parents.
- If any shared parents are found, we're removing them from the Sim, and then we apply the correct bits, and remake the FamilyTree.
- The source of truth for Sibling Sims comes from their parents, so we had to take some drastic measures to fix this retroactively. If we don't want to apply this fixup, there will be a need to override the fix from the Genealogy Tracker.
</Card>
